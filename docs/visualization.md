# Visualization

How to use the existing plot library and how to extend it.

## What's provided

`src/viz/plots.py` exposes 10+ functions producing publication-quality matplotlib + seaborn figures:

| Function | Inputs | Output |
|---|---|---|
| `plot_corpus_overview(df, path)` | labeled corpus DF | per-subreddit post counts + average lengths |
| `plot_length_distribution(df, path)` | labeled corpus DF | box plot of post length by subreddit |
| `plot_temporal(df, path)` | labeled corpus DF | stacked area chart of posts over time |
| `plot_label_distribution(df, path)` | labeled corpus DF | heatmap of weak-positive rate per (subreddit × label) |
| `plot_label_cooccurrence(df, path)` | labeled corpus DF | phi-coefficient matrix between labels |
| `plot_pr_roc(y_true, y_score, path)` | true labels + scores | ROC + PR curves side-by-side |
| `plot_calibration(y_true, y_score, path)` | true labels + scores | reliability diagram + score histogram |
| `plot_confusion(y_true, y_pred, path)` | true + hard preds | 2×2 confusion matrix |
| `plot_subreddit_f1(by_sub, path)` | DataFrame from `confusion_by_subgroup()` | per-subreddit F1 bars |
| `plot_linguistic_markers(markers, path)` | DataFrame from `compare_features_by_label()` | top-N Cohen's d bars with significance stars |

All functions:

- Take a path and write a PNG.
- Apply the project's seaborn style via `set_style()` (called automatically inside `runner.run_all`).
- Return the `Path` they wrote, so you can chain or log them.

## Generating everything

```bash
# After collection + preprocessing + labeling + training:
anxiety plot --run-dir experiments/runs/tfidf_logreg
```

Produces every figure into `docs/figures/` (configurable via `--figures-dir`).

## Style

`set_style()` configures:

```python
sns.set_theme(
    style="whitegrid",
    context="notebook",
    palette="colorblind",
    font_scale=1.05,
    rc={"figure.dpi": 110, "savefig.dpi": 130, ...}
)
```

If you want a different look (e.g. for a journal submission), call your own `sns.set_theme()` after importing `set_style`.

## Custom plots

Two patterns. Use whichever fits.

### Pattern A — call existing functions ad-hoc

```python
import pandas as pd
from src.viz.plots import plot_label_distribution, set_style

set_style()
df = pd.read_parquet("data/processed/labeled.parquet")
plot_label_distribution(df.query("subreddit != 'relationship_advice'"),
                        "docs/figures/label_distribution__no_ra.png")
```

### Pattern B — write a new function in `plots.py`

A reasonable template:

```python
def plot_my_thing(data: pd.DataFrame, out_path: str | Path, **opts) -> Path:
    """One-liner describing what's plotted."""
    fig, ax = plt.subplots(figsize=(10, 5))
    sns.lineplot(data=data, x=..., y=..., ax=ax)
    ax.set_xlabel("...")
    ax.set_ylabel("...")
    ax.set_title("...")
    return _save(fig, out_path)
```

Then add it to `runner.run_all` if you want it auto-generated by `anxiety plot`.

## Worked example: anxiety rate over time per subreddit

```python
import pandas as pd
import seaborn as sns
import matplotlib.pyplot as plt
from src.viz.plots import set_style

set_style()
df = pd.read_parquet("data/processed/labeled.parquet")
df["year"] = pd.to_datetime(df["created_utc"], unit="s").dt.year

rates = (
    df.assign(pos=(df["weak_anxiety"] >= 0.5).astype(int))
      .groupby(["year", "subreddit"])["pos"].mean()
      .unstack()
      .dropna(how="all")
)

fig, ax = plt.subplots(figsize=(12, 5))
rates.plot(ax=ax, marker="o", linewidth=2)
ax.set_ylabel("Weak anxiety-positive rate")
ax.set_title("Anxiety positive rate over time, by subreddit")
ax.legend(loc="upper left", ncol=2, frameon=True)
fig.savefig("docs/figures/custom__anxiety_over_time.png", bbox_inches="tight")
```

## Programmatic vs CLI

The CLI is a thin shell over `src.viz.runner.run_all`:

```python
from src.viz.runner import run_all

paths = run_all(
    figures_dir="docs/figures",
    labeled_path="data/processed/labeled.parquet",
    run_dir="experiments/runs/tfidf_logreg",
)
```

Use the programmatic path inside notebooks; use the CLI for batch runs and Makefile targets.

## Output convention

All figures go under `docs/figures/`. Naming:

- `<plot_name>.png` for corpus-level (e.g. `corpus_overview.png`).
- `<plot_name>__<target>.png` for target-specific (e.g. `pr_roc__health_anxiety.png`).

The naming is deliberate so README/docs can embed `![](docs/figures/X.png)` references that survive regeneration.
